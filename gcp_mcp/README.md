# GCP MCP Server (Commercial Properties)

An MCP (Model Context Protocol) server that provides tools to access commercial property data. It is built using FastMCP and is designed to run as a service (e.g., on Cloud Run).

## Features

-   **FastMCP Server**: Uses `fastmcp` to expose tools.
-   **Property Data**: Loads data from a local `listings.json` (generated by `gen_props.py`).
-   **Exposed Tools**:
    -   `search_properties_by_type`: Search for commercial properties by type (e.g., Office Building, Retail Space).
    -   `get_property_types`: List unique property types available.
    -   `get_property_details`: Get details for a specific property listing ID (e.g., CRE1001).

## Setup and Installation

The project uses `uv` for dependency management.

### Dependencies

Specified in `pyproject.toml`:
-   `fastmcp>=2.13.3`

### Setup

Make sure you have `uv` installed. Sync dependencies:

```bash
uv sync
```

### Running Locally

To run the MCP server locally (FastMCP will run over HTTP on port 8080 by default):

```bash
uv run server.py
```

## Deployment to Google Cloud Run

The server can be deployed to Google Cloud Run using the provided `deploy.sh` script:

```bash
./deploy.sh
```

The script executes `gcloud run deploy` with the following configuration:
-   **Region**: `europe-west1`
-   **Authentication**: Unauthenticated access is **disabled** (`--no-allow-unauthenticated`).
-   **Source**: Local directory (`.`).

Make sure you are authenticated with the Google Cloud SDK before running this script.
